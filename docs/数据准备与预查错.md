# 数据准备与预查错

本文档说明新数据分析流程中，原始数据如何放置、如何补写年月、如何合并多月数据，以及主流程预查错如何阻断并等待人工确认。

推荐主入口：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

---

## 1. 原始数据目录约定

主流程固定读取：

- `data/raw/{year}/{batch}`

示例：

- `data/raw/2026/3月`
- `data/raw/2026/1-2月`
- `data/raw/2026/Q1`

标准来源文件名由 `pipeline_paths.py` 定义：

- `展览.xlsx`
- `会议.xlsx`
- `酒店.xlsx`
- `餐饮.xlsx`
- `会展服务商.xlsx`
- `旅游.xlsx`

业务上允许某个批次只存在其中一部分来源文件；例如某个月没有酒店或旅游数据，是正常情况。只有当批次目录里一个标准来源文件都没有时，主流程才会阻断。

---

## 2. 单月批次与合并批次

### 2.1 单月批次

单月批次目录名形如：

- `1月`
- `2月`
- `3月`
- `12月`

判断逻辑在 `pipeline_paths.py` 的 `parse_single_month_batch()` 中，只识别 `1月` 到 `12月`。

单月批次如果缺少 `年份` / `月份` 列，主流程会自动补齐：

- `年份 = --year` 参数
- `月份 = 批次名中的月份数字`

例如：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

若 `data/raw/2026/3月` 中的来源文件缺少年月列，程序会补为：

- `年份 = 2026`
- `月份 = 3`

### 2.2 合并批次

合并批次目录名形如：

- `1-2月`
- `Q1`

这类目录里可能混有多个自然月的数据。程序无法从目录名判断每一行属于哪一个月，因此：

- 缺少 `年份` / `月份` 列时，主流程会阻断
- 不会自动补写年月
- 需要先人工准备好，或按“先单月补年月、再合并”的方式重新生成

当前流程中，`Q1` 是一个已经准备好的正式批次目录，不是运行时自动由 `1-2月 + 3月` 拼出来。

## 3. 预查错流程

预查错入口：

- `main_pipeline.py`
- `pipeline_runtime.py`
- `pipeline_precheck.py`

执行主流程后，程序会先调用 `run_precheck()`：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

### 3.1 会阻断的情况

预查错发现以下问题会阻断：

- 原始批次目录不存在
- 批次目录里完全没有标准来源文件
- 标准来源文件缺少 `问卷数据` sheet
- Excel 文件读取失败
- 合并批次缺少 `年份` / `月份` 列
- 存在未映射客户标签记录

### 3.2 不会阻断的情况

以下情况不会作为异常提示或阻断：

- 只缺少某几个标准来源文件
- 某个月份没有某些客户分组数据
- 单月批次缺少 `年份` / `月份` 列

单月批次缺少年月时，预查错只记录 warning，后续由主流程自动补齐。

### 3.3 阻断后如何继续

如果存在阻断问题，主流程会停下来提示修改原始数据目录。修改完成后，在终端输入以下任一内容继续：

- `y`
- `yes`
- `继续`

如果输入以下任一内容，主流程终止：

- `stop`
- `quit`
- `exit`

确认继续后，程序会重新执行预查错；只有预查错通过后，才进入分项统计、汇总、样本统计和 PPT 生成。

### 3.4 预查错日志

主流程日志固定写入：

- `logs/pipeline/{year}/{batch}/precheck.log`
- `logs/pipeline/{year}/{batch}/pipeline.log`
- `logs/pipeline/{year}/{batch}/unmapped_customer_records.log`

## 4. 未映射客户记录核查

未映射记录检查由 `check_unmapped_customer_records.py` 执行，主流程会在预查错中自动调用。

检查口径：

- 规则来源：`survey_customer_category_rules.py`
- 只检查目录模式启用的规则，即 `directory_enabled=True`
- 按规则中的 `source_file_name` 定位来源文件
- 若规则要求 `auxiliary_column + data_column`，则按“辅助标签 + 数据标签组合”匹配
- 若规则只要求 `data_column`，则只按数据标签匹配

能发现的问题包括：

- 数据标签不在映射规则中
- 辅助标签 + 数据标签组合不在映射规则中
- 数据标签存在但辅助标签为空，导致无法命中规则

单独运行示例：

```bash
uv run python check_unmapped_customer_records.py \
  --input-dir 'data/raw/2026/1-2月' \
  --log-file 'logs/pipeline/2026/1-2月/manual_unmapped_check.log'
```

常用参数：

- `--input-dir`：要检查的原始数据目录
- `--sheet-name`：目标 sheet，默认 `问卷数据`
- `--log-file`：日志输出路径

## 5. 年份 / 月份列补写

年月补写脚本：

- `fill_year_month_columns.py`

主流程在单月批次中自动调用，也可以手动运行。

基本用法：

```bash
uv run python fill_year_month_columns.py \
  --input-dir 'data/raw/2026/3月' \
  --year '2026' \
  --month '3'
```

常用参数：

- `--input-dir`：要处理的目录
- `--year`：写入 `年份` 列的文本值
- `--month`：写入 `月份` 列的文本值
- `--recursive`：递归处理子目录
- `--sheet-name`：目标 sheet，默认 `问卷数据`

行为说明：

- 只处理 `.xlsx`
- 跳过 Excel 临时文件
- 只修改目标 sheet
- 从第 2 行开始写入，第 1 行视为表头
- 如果 `年份` / `月份` 列已存在，则覆盖原值
- 如果列不存在，则追加到表头末尾
- 如果文件缺少目标 sheet，则跳过并在输出中提示

重要边界：

- 该脚本是整文件批量写值，不会逐行识别月份
- 适合处理单月目录
- 不适合直接处理 `1-2月`、`Q1` 这类混合月份目录

## 6. 多月数据合并

多月合并脚本：

- `merge_questionnaire_workbooks.py`

推荐顺序：

1. 先分别给每个单月目录补写 `年份` / `月份`
2. 再合并同名 Excel
3. 把合并结果输出到 `data/raw/{year}/{batch}`
4. 最后运行 `main_pipeline.py`

示例：合并为 `Q1`

```bash
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/1月' --year '2026' --month '1'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/2月' --year '2026' --month '2'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/3月' --year '2026' --month '3'

uv run python merge_questionnaire_workbooks.py \
  --input-dir 'data/raw/2026/1月' \
  --input-dir 'data/raw/2026/2月' \
  --input-dir 'data/raw/2026/3月' \
  --output-dir 'data/raw/2026/Q1'
```

合并规则：

- 按文件名分组，例如多个目录中的 `餐饮.xlsx` 合并成一个 `餐饮.xlsx`
- 只读取 `问卷数据` sheet
- 输出文件只包含一个 `问卷数据` sheet
- 同名文件先按列语义对齐，再按行追加
- `Q1-调研类别`、`Q2-调研类别` 会忽略题号前缀后视为同一列
- 第一个文件的列顺序作为基础，后续新增列追加到末尾
- 某个文件缺少的列，对应数据行留空

会跳过的情况：

- 同名文件中任意一个缺少目标 sheet
- 文件自身存在重复列名
- 忽略题号前缀后出现语义重复列名

不会自动做的事情：

- 不按 `提交序号` 去重
- 不自动保留最新记录
- 不自动修正客户标签
- 不自动处理标签冲突

## 7. 问卷期次列预处理

部分新版问卷会在 `问卷数据` sheet 第三列新增期次列，例如：

- `一期`
- `二期`
- `三期`

这会导致依赖固定列位的标签列和指标列右移。处理脚本是：

- `phase_column_preprocess.py`

当前 `survey_stats.py` 在正式统计前会自动调用该预处理。

处理规则：

- 第三列没有检测到 `X期` 特征时，不做移动
- 第三列检测到 `一期`、`二期` 等期次标记时，将整列移动到最后一列
- 移动后直接保存回原始 Excel，再继续统计
- 如果其他列检测到期次标记，会在单独运行脚本时提示“可能已经处理过”

单独运行示例：

```bash
uv run python phase_column_preprocess.py 'data/raw/2026/3月/展览.xlsx'
```

一次处理多个文件：

```bash
uv run python phase_column_preprocess.py data/raw/2026/3月/*.xlsx
```

指定 sheet：

```bash
uv run python phase_column_preprocess.py 'data/raw/2026/3月/展览.xlsx' --sheet-name '问卷数据'
```

## 8. 推荐操作模板

### 单月直接跑

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

适合原始数据已经放在 `data/raw/2026/3月` 的场景。

### 已准备好的合并批次

```bash
uv run python main_pipeline.py --year 2026 --batch 1-2月
uv run python main_pipeline.py --year 2026 --batch Q1
```

前提是对应目录已经存在，并且合并批次中的 `年份` / `月份` 信息已经正确存在。

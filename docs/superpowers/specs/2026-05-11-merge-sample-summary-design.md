# 交互式合并样本统计设计

## 背景

当前主流程 `main_pipeline.py` 面向已经准备好的单个批次目录运行，会执行预查错、满意度分项统计、满意度汇总、样本统计和 PPT 生成。用户现在需要一个更窄的工具：从 `data/raw/{year}` 下选择多个已有来源目录，合并原始问卷数据，并只生成分月样本统计汇总表。

仓库已有可复用能力：

- `merge_questionnaire_workbooks.py`：按同名 Excel 文件合并多个来源目录，合并 `问卷数据` sheet，并按忽略题号前缀后的语义表头对齐列。
- `fill_year_month_columns.py`：对单月来源目录批量补写或覆盖 `年份`、`月份` 列。
- `sample_table.py`：按客户类型映射规则生成客户类型样本统计表，且已支持从 `年份`、`月份` 列自动生成分月统计列。

## 目标

新增独立脚本 `merge_sample_summary.py`，运行方式：

```bash
uv run python merge_sample_summary.py --year 2026
```

脚本完成：

1. 列出 `data/raw/{year}` 下所有直接子文件夹。
2. 通过终端 TUI 让用户多选来源目录。
3. 让用户输入合并后的输出批次名，例如 `Q1`。
4. 必要时给明确单月来源目录补写 `年份`、`月份`。
5. 合并选中的来源目录到 `data/raw/{year}/{batch_name}`。
6. 只生成客户类型样本统计汇总表到 `data/sample_summary/{year}/{batch_name}/{batch_name}客户类型样本统计表.xlsx`。

## 非目标

- 不生成满意度分项统计。
- 不生成满意度汇总表。
- 不生成 PPT。
- 不改变现有 `main_pipeline.py --year --batch` 的行为。
- 不改变样本统计的客户类型映射规则、目标样本量和表格样式。
- 不对合并数据按提交序号去重，也不自动判断记录新旧版本。

## 交互设计

脚本读取 `data/raw/{year}` 下所有直接子文件夹，并进入终端选择界面。

TUI 按键：

- `↑` / `↓`：移动光标。
- `Space`：勾选或取消当前目录。
- `Enter`：确认选择。
- `q` 或 `Esc`：退出。

显示规则：

- 选中目录显示为 `[x]`。
- 未选中目录显示为 `[ ]`。
- 当前光标行高亮显示。

如果当前终端不支持 TUI，脚本降级为编号输入模式。编号模式支持逗号和范围，例如 `1,2`、`1-3`。

确认来源目录后，用户输入输出批次名。输出批次名必须是普通目录名，不能为空，不能包含路径分隔符，不能与任一选中来源目录同名。

## 目录行为

输入来源根目录：

- `data/raw/{year}`

来源目录：

- `data/raw/{year}` 下所有直接子文件夹都会列出供用户选择。
- 允许选择 `1月`、`3月` 等明确单月目录。
- 允许选择 `1-2月` 等已有混合目录。
- 不额外限制目录名是否符合月份或季度格式。

输出路径：

- 合并原始数据：`data/raw/{year}/{batch_name}`
- 样本统计目录：`data/sample_summary/{year}/{batch_name}`
- 样本统计文件：`data/sample_summary/{year}/{batch_name}/{batch_name}客户类型样本统计表.xlsx`

如果输出批次目录或样本统计表已经存在，脚本会询问是否覆盖。用户确认后，只覆盖本工具生成的合并原始 Excel 文件和目标样本统计表，不删除 `data/satisfaction_detail`、`data/satisfaction_summary`、`data/ppt` 等其他主流程输出。

## 数据处理流程

脚本确认输入后按以下顺序执行：

1. 解析和校验 `--year`。
2. 发现 `data/raw/{year}` 下的直接子文件夹。
3. 用户多选来源目录。
4. 用户输入输出批次名。
5. 检查输出批次是否已存在，必要时询问覆盖。
6. 对选中的来源目录逐个判断是否为明确单月目录。
7. 对 `1月` 到 `12月` 这种明确单月目录，合并前调用现有年月补写逻辑，写入 `年份 = year`、`月份 = 月份数字`。
8. 对 `1-2月` 这类混合目录，不自动补写年月，只检查其中参与统计的问卷文件是否具备 `年份`、`月份` 列。
9. 如果任一混合来源目录缺少 `年份` 或 `月份` 列，脚本阻断并提示用户先修复来源数据。
10. 调用现有合并逻辑，把选中目录按文件名合并到 `data/raw/{year}/{batch_name}`。
11. 如果合并结果中存在读取失败、缺少 `问卷数据` sheet、重复列名或语义重复列名等失败项，脚本打印合并明细并停止。
12. 合并成功后，调用现有样本统计逻辑生成客户类型样本统计表。

样本统计表基于合并后数据中的 `年份`、`月份` 列自动发现分月列。例如合并数据中存在 `1-2` 和 `3`，表头将包含 `1-2月`、`3月`。

## 年月规则

明确单月目录：

- 只识别 `1月` 到 `12月`，以及现有 `parse_single_month_batch()` 支持的等价写法。
- 这类来源目录可以自动补写或覆盖 `年份`、`月份`。

混合来源目录：

- 例如 `1-2月`。
- 允许作为来源参与合并。
- 不自动补写年月。
- 必须已经包含正确的 `年份`、`月份` 列。

这样可以避免把混合数据整批错误写成同一个月份。

## 异常处理

- `data/raw/{year}` 不存在：报错并退出。
- 年份目录下没有直接子文件夹：报错并退出。
- 用户未选择任何来源目录：提示必须至少选择一个，并返回选择界面。
- 输出批次名为空、包含路径分隔符或与来源目录冲突：拒绝并重新输入。
- 输出批次已存在且用户拒绝覆盖：退出，不写入新结果。
- 混合来源目录缺少 `年份` 或 `月份` 列：阻断，不合并。
- 合并阶段出现失败项：打印合并明细，不生成样本统计表。
- 样本统计失败：保留已生成的合并原始数据，并报告错误。

## 测试策略

采用测试驱动开发。优先测试可拆分的纯逻辑和流程编排，不直接断言 curses 画面细节。

测试覆盖：

- 发现并排序 `data/raw/{year}` 下的直接子文件夹。
- 输出批次名校验。
- 编号 fallback 输入解析。
- 明确单月目录触发年月补写。
- `1-2月` 等混合目录不触发自动补写。
- 混合目录缺少 `年份` 或 `月份` 列时阻断。
- 合并成功后只调用样本统计生成，不调用满意度分项、满意度汇总或 PPT 生成。
- 合并失败时不生成样本统计。
- 输出已存在时的覆盖确认分支。

测试命令使用 `uv run`，例如：

```bash
uv run python -m unittest tests/test_merge_sample_summary.py
```

## 验收标准

- 用户可以运行 `uv run python merge_sample_summary.py --year 2026` 并通过终端界面选择多个来源目录。
- 选择 `1-2月`、`3月` 并输入 `Q1` 后，合并原始数据写入 `data/raw/2026/Q1`。
- 只生成 `data/sample_summary/2026/Q1/Q1客户类型样本统计表.xlsx`。
- 不生成或改写满意度分项、满意度汇总、PPT 输出。
- 单月来源目录缺少年月列时可自动补齐。
- 混合来源目录缺少年月列时会阻断并提示修复。
- 相关单元测试通过。

# survey_stats 客户类别规则说明

## 目的

`survey_stats.py` 的目录模式现在按一套显式的客户类别规则运行，而不是只依赖模板内零散的 `role_name` / `row_conditions`。

规则定义文件：

- [survey_customer_category_rules.py](/Users/zhangqijin/PycharmProjects/hangbo/survey_customer_category_rules.py)

## 规则口径

每条客户类别规则至少包含：

- `sequence_number`：来自 `datas/customer_mapping.xlsx` 的“序号”列，用于固定后续展示顺序
- `source_file_name`：来源文件名
- `template_name`：沿用的统计模板
- `data_column`：数据标签所在列，按 Excel 列字母表示
- `data_values`：数据标签值，支持多个值并集
- `auxiliary_column`：辅助标签所在列
- `auxiliary_values`：辅助标签值

当前目录模式严格按 `客户类别对照表V1.0-补充标签所在列.xlsx` 的 V1.0 口径执行。

其中：

- 正式客户类别规则会记录 `sequence_number`
- 内部聚合组件规则（如 `酒店宴会`、`酒店自助餐`）不占用展示序号，`sequence_number` 为空
- 如果后续需要按固定展示顺序输出客户类别，可直接使用 `DISPLAY_ORDERED_CUSTOMER_CATEGORY_RULES`

## 当前实现约定

- 展览类客户：按 `C=展览` + `E=具体身份标签` 识别
- 会议类客户：按 `C=会议/酒店会议` + `E=具体身份标签` 识别
- 餐饮类客户：按 `C=餐饮/酒店餐饮` + `D=具体餐饮标签` 识别
- 旅游类客户：按 `C=旅游客户身份标签` 识别
- 酒店类客户：按 `C=酒店客户身份标签` 识别
- 会展服务商：按 `D=会展服务商` 识别

## 酒店餐饮客户

`酒店餐饮客户` 不是单一模板，而是聚合结果：

- 组件一：`酒店宴会`
- 组件二：`酒店自助餐`
- 组件三：`商务简餐`
- 组件四：`宴会`

目录模式会先分别按以下规则计算四份内部结果：

- `C=酒店餐饮` + `D=酒店宴会`
- `C=酒店餐饮` + `D=酒店自助餐`
- `C=酒店餐饮` + `D=商务简餐`
- `C=酒店餐饮` + `D=宴会`

然后对这些内部统计结果按列做算术平均，生成最终的 `酒店餐饮客户`。

## 酒店参会客户

`酒店参会客户` 按以下并集口径识别：

- `C=酒店会议` + `E=酒店参会客户`
- `C=酒店会议` + `E=参会人员`

## 表外组合处理

当前实现对表外组合采取 V1.0 严格口径：

- 不纳入任何客户类别统计
- 在目录模式结束后单独打印提示

当前仍按“辅助标签 + 数据标签”的精确组合判断归属；未在映射表中声明的跨组合仍会单独报出。

例如当前仍会排除这类组合：

- `会议.xlsx`：`会议 + 酒店参会客户`
- `餐饮.xlsx`：`餐饮 + 酒店宴会`、`餐饮 + 酒店自助餐`

## 输出命名

当前为了兼容已有流程，目录模式输出名仍优先沿用内部客户名：

- 例如 `会议主承办.xlsx`、`参会人员.xlsx`、`酒店会议主承办.xlsx`

新增聚合类别输出为：

- `酒店餐饮客户.xlsx`

如果后续要把输出文件名整体切到对照表里的业务名称，需要同步调整依赖这些文件名的后续模块。

# 未映射客户记录核查

`check_unmapped_customer_records.py` 用来检查原始问卷目录中，哪些记录没有被客户类别映射规则覆盖。

## 规则口径

- 规则来源固定为 `survey_customer_category_rules.py`
- 只按目录模式启用的规则检查，也就是 `directory_enabled=True` 的规则
- 扫描文件范围与 `survey_stats.py` 的目录模式一致，按规则中的 `source_file_name` 定位
- 如果规则同时要求 `auxiliary_column + data_column`，则按“辅助标签 + 数据标签组合”判定
- 如果规则只要求 `data_column`，则按“数据标签”判定

## 能查出的情况

- 某条记录的 `数据标签` 不在映射规则里
- 某条记录的 `辅助标签 + 数据标签` 组合不在映射规则里
- 某条记录有数据标签，但辅助标签为空，导致无法命中映射

## 输出内容

终端和日志文件内容一致，默认日志会写到 `/logs`，文件名形如：

```text
logs/unmapped_customer_records_YYYYMMDD_HHMMSS.log
```

输出分三部分：

1. 摘要
2. 未映射记录明细
3. 附加提示

明细中会列出：

- 来源文件名
- Excel 行号
- 辅助标签列和值
- 数据标签列和值
- 未命中映射的原因

## 用法

检查 `datas/1-2月`：

```bash
uv run python check_unmapped_customer_records.py \
  --input-dir './datas/1-2月'
```

指定日志文件：

```bash
uv run python check_unmapped_customer_records.py \
  --input-dir './datas/1-2月' \
  --log-file './logs/1-2月未映射记录核查.log'
```

指定 sheet：

```bash
uv run python check_unmapped_customer_records.py \
  --input-dir './datas/1-2月' \
  --sheet-name '问卷数据'
```

## 说明

- 当前脚本只检查规则定义里要求的标准来源文件
- 如果目录里缺少某些规则来源文件，会在“附加提示”里列出
- 如果没有发现异常，会明确输出“未发现未映射记录”

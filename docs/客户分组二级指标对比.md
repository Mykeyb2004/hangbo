# 客户分组二级指标对比

## 结论先看

- `summary_table.py` 不是简单把 `survey_stats.py` 的二级指标原样搬到汇总表。
- 对酒店散客/住宿团队、餐饮客户各组来说，二级指标维度基本一致，只是列名有统一改写。
- 对会展类、会议类、酒店会议类、酒店参会类来说，不是完全一致：汇总表中的 `餐饮服务` 列并不是取二级指标，而是回退去取三级指标 `餐饮服务`。
- 汇总表里的灰色单元格，代码层面表示“这个客户分组不适用这个列，不参与取值”。对于有 `survey_stats.py` 模板的客户分组，这些灰色列在 `survey_stats.py` 中也确实没有对应二级指标，等价于跳过。
- `survey_stats.py` 里有 `旅游团餐`，但 `summary_table.py` 没有对应行。
- `summary_table.py` 里有 `G20峰会体验馆` 和 `专项调研` 行，但 `survey_stats.py` 里没有对应模板。

## 灰色单元格到底代表什么

- 灰色单元格不是“有这个维度但没算出值”，而是“这个客户分组根本不适用这个列”。
- 在 `summary_table.py` 中，只要某列不在 `row.applicable_columns` 里，就直接涂灰，不参与取值。
- 在 `survey_stats.py` 中，是否参与计算取决于 `RoleDefinition.sections` 里有没有这个二级指标；没有定义就不会进入计算，也不会写进导出报表。

## 对这次对比最值得关注的点

1. 真正的不一致，集中在会展/会议/酒店会议/酒店参会这 8 类客户的 `餐饮服务` 列。
2. 这 8 类在 `survey_stats.py` 中只有二级指标 `配套服务`，并没有二级指标 `餐饮服务`。
3. 但 `summary_table.py` 为了出汇总表，额外把三级指标 `餐饮服务` 单独拉出来放成一列，所以汇总表里的这列不是“二级指标口径”。
4. 餐饮类、酒店散客/住宿团队的灰格处理，与 `survey_stats.py` 的模板定义是一致的。

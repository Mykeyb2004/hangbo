- 关闭窗口时自动保存最近一次会话，重新打开后恢复到上次编辑状态

- 扫描结果里建议提前显示“期次列疑似”“年月列是否存在”，减少用户反复跳页

### 页面右侧日志抽屉

```text
+--------------------------------------+
| 分项统计日志                         |
|--------------------------------------|
| [1/16] 正在处理：展览.xlsx（展览主承办）|
| [1/16] 已执行预处理：期次列已移动     |
| [1/16] 结果已保存 -> 展览主承办.xlsx  |
| [2/16] 正在处理：展览.xlsx（参展商）  |
| ...                                  |
+--------------------------------------+
```

### 交互建议

- `dry-run` 后如果没问题，自动提示“可以直接正式执行”
- 扫描后默认自动勾选全部 `可生成` 客群
- 用户可以通过 `全选 / 清空 / 仅选可生成` 或双击单行切换勾选状态
- 正式执行和 `dry-run` 都只跑当前已勾选客群，对应 `survey_stats.py --job ...`
- 如果某个客群因“来源文件存在但未匹配身份值”而跳过，应支持点击查看该来源文件实际识别到的身份值
- 分项统计完成后，页面底部出现两个快捷按钮：
  - `去生成汇总表`
  - `去生成PPT`

## 5. 一键执行主流程弹窗

### 页面目标

- 给不想逐页操作的用户一个“主流程自动执行”入口
- 但要允许用户控制是否包含可选步骤

### 草图

```text
+------------------------------------------------------------------------------------------------------+
| 一键执行主流程                                                                                       |
+------------------------------------------------------------------------------------------------------+
| 当前批次：2026年3月                                                                                  |
| 数据处理模式：单月处理                                                                               |
| 原始目录：/datas/3月                                                                                 |
|------------------------------------------------------------------------------------------------------|
| 执行步骤                                                                                             |
| [x] 预处理：兼容新版调查问卷数据结构                                                                 |
| [ ] 预处理：在数据源中加入年份+月份                                                                  |
| [x] 分项统计                                                                                         |
| [x] 汇总统计                                                                                         |
| [x] 生成PPT                                                                                          |
|------------------------------------------------------------------------------------------------------|
| 输出路径                                                                                             |
| 分项结果：/输出结果/3月                                                                              |
| 汇总结果：/汇总结果/3月                                                                              |

```

### 执行中的状态弹窗

```text
+------------------------------------------------------------------------------------------------------+
| 正在执行主流程                                                                                       |
+------------------------------------------------------------------------------------------------------+
| 总状态：执行中 2/4；当前步骤：生成分项统计                                                           |
|------------------------------------------------------------------------------------------------------|
| 步骤                     | 状态      | 进度明细                                                     |
| 1. 兼容新版调查问卷数据结构 | 已完成    | 执行完成，可继续下一步                                       |
| 2. 生成分项统计         | 执行中    | python survey_stats.py --config ... --job 展览主承办 ...     |
| 3. 生成汇总表           | 待执行    | 等待前置步骤                                                 |
| 4. 生成PPT              | 待执行    | 等待前置步骤                                                 |
|------------------------------------------------------------------------------------------------------|
| [查看详细日志]                                                                                       |
|------------------------------------------------------------------------------------------------------|
| [后台运行]                                                             [终止当前任务]              |
+------------------------------------------------------------------------------------------------------+
```

### 当前实现约定

- 主流程结束后，界面自动跳转到最后一个步骤对应页面
- 如果用户点击 `后台运行`，弹窗会隐藏，但任务继续执行
- 如果用户点击 `终止当前任务`，当前步骤会先进入“正在终止”，随后落到“已取消”

## 6. 推荐的视觉层级

### 一级按钮

- `一键执行主流程`
- `开始生成分项统计`
- `生成汇总表`
- `生成PPT`

### 二级按钮

- `扫描`
- `dry-run`
- `保存配置`
- `打开输出目录`

### 状态标签

- `成功`
- `待执行`
- `已完成`
- `跳过`
- `异常`
- `缺少来源文件`
- `未匹配身份值`

---

## 7. 推荐的首页卡片文案

为了让业务同事更容易理解，建议少出现脚本名，多出现业务名：

- `导入数据源`
- `整理原始问卷`
- `生成分项统计`
- `生成客户类型汇总表`
- `生成汇报PPT`

脚本名可以只出现在“高级设置”或“日志详情”里。

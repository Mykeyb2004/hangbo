# 非月份满意度均值的解释



可以解释，但要先说结论：

当前算法“有统计学上可识别的口径”，不是乱算；但它也不是“唯一权威标准”，更像一种明确的业务聚合规则。  
它对应的是一种“分层、等权”的综合评分法，不是经典抽样统计里唯一推荐的总体估计量。

**当前算法到底在算什么**

代码里现在是两层等权平均：

- 在 [survey_stats.py](/Users/zhangqijin/PycharmProjects/hangbo/survey_stats.py#L1163) 里：
  - 先算每个三级指标的均值
  - 再把三级指标等权平均成 section 分
  - 再把各个 section 等权平均成该客户类型的“总分”
- 在 [summary_table.py](/Users/zhangqijin/PycharmProjects/hangbo/summary_table.py#L511) 里：
  - 再把各客户类型的“总分”做一次等权平均，得到汇总表底部的“总分”

所以它本质上是：

```text
指标等权 -> 二级维度等权 -> 客户类型等权
```

这在统计/评价领域可以对应到两个常见概念：

- `composite indicator / 综合指数`
- `macro average / 宏平均`

也就是说，它回答的问题更像是：

```text
“如果把每个客户类型都当成同样重要的业务单元，
那整体盘面的满意度是多少？”
```

而不是：

```text
“如果随机抽一个Q1客户，
他的平均满意度是多少？”
```

这两个问题不是一回事。

**为什么 Q1 不是 1-2月 和 3月 的简单平均**

因为当前 Q1 不是按“月份等权”算的，而是按“Q1 合并后的原始样本重新计算每个客户类型，再对客户类型等权平均”算的。

所以它不是：

```text
(9.73 + 9.57) / 2
```

而是更接近：

```text
平均(Q1里每个客户类型的Q1总分)
```

再加上：
- `1-2月` 有值的客户类型只有 `11` 个
- `3月` 有值的客户类型有 `17` 个
- `Q1` 有值的客户类型也有 `17` 个

因此三张表的“总分”分母都不一样，当然不会满足“季度 = 两个月均值”。

**这套算法有没有理论支持**

有“方法论上的支持”，但没有“必须这样算”的唯一权威。

可以这样理解：

1. 如果你的目标是“各客户类型一视同仁的管理看板”
   那当前算法是合理的。
   因为它避免了大样本客户类型把小样本客户类型完全淹没。

2. 如果你的目标是“所有受访者整体平均满意度”
   那当前算法就不是最标准的。
   更标准的是：
   - 按样本量加权的平均
   - 如果有抽样权重，还应使用调查权重

3. 如果你的目标是“季度值应该和月度报表可直接对齐”
   那就要先定义季度口径：
   - 月份等权
   - 样本量加权
   - 客户类型等权
   这三种都可以“有理有据”，但结果会不同。

**所以，这套算法的定位应该怎么说**

最准确的说法是：

- 它不是经典“总体均值估计”的唯一权威算法
- 它是一种“层级等权综合评分法”
- 适合做客户类型公平展示的经营分析
- 不适合直接解释成“所有样本的总体平均满意度”

**我建议**

如果你后面要对外解释，最好把口径写清楚：

- `当前总分 = 客户类型等权平均后的综合满意度`
- 不是 `按月份平均`
- 也不是 `按全部样本量加权平均`

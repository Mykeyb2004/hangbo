# 统计口径与结果说明

本文档说明新流程中满意度分项统计、满意度汇总表和样本统计表的输入、输出、计算口径与关键参数。

主流程顺序：

```text
data/raw/{year}/{batch}
  -> data/satisfaction_detail/{year}/{batch}
  -> data/satisfaction_summary/{year}/{batch}
  -> data/sample_summary/{year}/{batch}
```

---

## 1. 满意度分项统计

脚本入口：

- `survey_stats.py`

主流程调用位置：

- `pipeline_runtime.py` 中的 `run_directory_batch()`

输入：

- `data/raw/{year}/{batch}`

输出：

- `data/satisfaction_detail/{year}/{batch}`

输出内容：

- 每个客户类型一份 `xlsx`
- 默认列包含 `指标`、`满意度`、`重要性`
- 总体行、一级指标行、明细指标行会用不同样式区分，后续汇总表和 PPT 会复用这些结构

主流程默认参数来自 `pipeline.defaults.toml`：

```toml
sheet_name = "问卷数据"
calculation_mode = "template"
```

---

## 2. 客户类别规则

客户类别规则定义在：

- `survey_customer_category_rules.py`

规则主要包含：

- `source_file_name`：来源文件名
- `template_name`：使用的统计模板
- `customer_group`：客户大类
- `customer_category`：展示用客户类型
- `data_column`：数据标签列
- `data_values`：数据标签值
- `auxiliary_column`：辅助标签列
- `auxiliary_values`：辅助标签值
- `directory_enabled`：是否参与目录模式统计

当前主要识别口径：

- 展览类客户：`C=展览` + `E=具体身份标签`
- 会议类客户：`C=会议/酒店会议` + `E=具体身份标签`
- 餐饮类客户：`C=餐饮/酒店餐饮` + `D=具体餐饮标签`
- 旅游类客户：`C=旅游客户身份标签`
- 酒店类客户：`C=酒店客户身份标签`
- 会展服务商：`D=会展服务商`

### 2.1 酒店餐饮客户

`酒店餐饮客户` 是聚合客户类型，不是单一来源模板。

内部组件包括：

- `酒店宴会`
- `酒店自助餐`
- `商务简餐`
- `宴会`

目录模式会先分别计算内部组件，再按列做算术平均，生成最终的 `酒店餐饮客户.xlsx`。

### 2.2 酒店参会客户

`酒店参会客户` 使用并集口径：

- `C=酒店会议` + `E=酒店参会客户`
- `C=酒店会议` + `E=参会人员`

### 2.3 表外组合

未在规则中声明的辅助标签 + 数据标签组合，不纳入任何客户类别统计。

这类记录会在预查错阶段由 `check_unmapped_customer_records.py` 检出，阻断主流程，避免脏标签进入统计结果。

## 3. 计算模式

`survey_stats.py` 支持两种计算模式：

- `template`
- `summary`

新主流程默认使用：

```toml
calculation_mode = "template"
```

### 3.1 `template` 模式

`template` 是当前推荐和默认模式。

特点：

- 按 `survey_stats.py` 模板定义的原始二级指标计算
- 保持原分项统计模板口径不变
- 输出适合作为 PPT 的明细页数据源
- 汇总表的总分直接读取分项文件中的总体满意度

### 3.2 `summary` 模式

`summary` 是按汇总表展示维度重组的口径。

典型变化：

- `会展服务` / `会场服务` 重组为 `产品服务`
- `入住服务` 重组为 `产品服务`
- `智慧场馆` / `智慧服务` 重组为 `智慧场馆/服务`
- 会展和酒店会议类会把 `餐饮服务` 从 `配套服务` 中拆出
- 餐饮类只保留汇总表非灰格维度

说明：

- `summary` 模式会改变二级指标结构
- 总体分、二级指标分和导出行顺序都可能不同于 `template`
- 除非需要专门验证汇总维度口径，否则新主流程不建议切换

单独运行时可临时覆盖：

```bash
uv run python survey_stats.py --config job.toml --calculation-mode summary
```

## 4. 满意度汇总表

脚本入口：

- `summary_table.py`

主流程调用位置：

- `pipeline_runtime.py` 中的 `generate_summary_report()`

输入：

- `data/satisfaction_detail/{year}/{batch}`

输出：

- `data/satisfaction_summary/{year}/{batch}/{batch}客户类型满意度汇总表.xlsx`

例如：

- `data/satisfaction_summary/2026/3月/3月客户类型满意度汇总表.xlsx`
- `data/satisfaction_summary/2026/Q1/Q1客户类型满意度汇总表.xlsx`

汇总表标题：

- `杭博客户类型满意度情况表`

### 4.1 汇总列

汇总表固定输出以下指标列：

- `总分`
- `产品服务`
- `硬件设施`
- `配套服务`
- `智慧场馆/服务`
- `餐饮服务`

### 4.2 客户大类顺序

行顺序主要来自 `survey_customer_category_rules.py` 中的展示顺序。

当前大类包括：

- `一、会展客户`
- `二、餐饮客户`
- `三、G20峰会体验馆`
- `四、专项调研`
- `五、酒店客户`

其中：

- `四、专项调研` 保留为空白占位行
- 不适用的灰格位置显示 `--`
- 其他空值保持空白

### 4.3 取数规则

汇总表读取分项统计文件：

- 总分：读取总体满意度
- 产品服务：
  - 会展/会议/酒店会议类优先取 `会展服务`，兼容 `会场服务`
  - 酒店散客、住宿团队取 `入住服务`
  - G20 取 `旅游服务`
- 硬件设施：取一级指标 `硬件设施`
- 配套服务：取一级指标 `配套服务`
- 智慧场馆/服务：优先取 `智慧场馆`，兼容 `智慧服务`
- 餐饮服务：优先取一级指标 `餐饮服务`，没有时回退到明细指标 `餐饮服务`

底部 `总分` 行按各样本类型行做列平均，空值不参与计算。

### 4.4 单独运行

```bash
uv run python summary_table.py \
  --input-dir data/satisfaction_detail/2026/Q1 \
  --output-dir data/satisfaction_summary/2026/Q1 \
  --output-name Q1客户类型满意度汇总表.xlsx
```

常用参数：

- `--input-dir`：分项统计 `xlsx` 所在目录
- `--output-dir`：汇总表输出目录
- `--output-name`：输出文件名
- `--recursive`：递归扫描输入目录

## 5. 样本统计表

脚本入口：

- `sample_table.py`

主流程调用位置：

- `pipeline_runtime.py` 中的 `generate_sample_table_report()`

输入：

- `data/raw/{year}/{batch}`

输出：

- `data/sample_summary/{year}/{batch}/{batch}客户类型样本统计表.xlsx`

例如：

- `data/sample_summary/2026/3月/3月客户类型样本统计表.xlsx`
- `data/sample_summary/2026/Q1/Q1客户类型样本统计表.xlsx`

默认配置：

- `sample_table.default.toml`

配置内容包括：

- 表头标题
- 输出 sheet 名
- 默认输出文件名
- 客户大类与样本类型展示顺序
- 每一行的目标样本量
- 每一行对应的映射规则名
- 特殊行的手工覆盖值

### 5.1 样本统计口径

样本统计直接读取原始问卷的 `问卷数据` sheet。

统计逻辑：

- 映射规则复用 `survey_customer_category_rules.py`
- 实际样本数来自命中规则的原始记录数
- 月份分组基于原始数据中的 `年份` + `月份`
- 如果 `年份` 缺失或为空，默认使用主流程传入的 `--year`
- 如果没有显式传入月份分组参数，会自动按实际出现的年月组合生成分组列

自动分组列展示规则：

- 同一年内显示为 `1月`、`2月`、`3月`
- 存在多个年份或非默认年份时，会带年份显示

### 5.2 特殊行

当前样本统计表保留以下特殊业务行：

- `会展流失主办客户`：实际样本数固定记 `0`
- `酒店散客`：实际样本数复用 `散客` 的映射结果
- `四、专项调研`：手工维护行，位于 G20 与酒店客户之间
- `六、酒店暗访（次）`：手工维护行，位于合计之前

手工维护行的 `总执行样本量` 使用 Excel 公式汇总所有月份列，方便后续人工编辑。

### 5.3 表格结构

输出列包括：

- `客户大类`
- `样本类型`
- `样本量`
- `样本进度百分比`
- `总执行样本量`
- 动态月份分组列

示例：

```text
客户大类 | 样本类型 | 样本量 | 样本进度百分比 | 总执行样本量 | 1月 | 2月 | 3月
```

每个客户大类后会追加小计行，最后追加合计行。

### 5.4 单独运行

```bash
uv run python sample_table.py \
  --input-dir data/raw/2026/Q1 \
  --output-dir data/sample_summary/2026/Q1 \
  --output-name Q1客户类型样本统计表.xlsx
```

指定配置：

```bash
uv run python sample_table.py \
  --input-dir data/raw/2026/Q1 \
  --output-dir data/sample_summary/2026/Q1 \
  --config sample_table.default.toml
```

手动指定月份分组列：

```bash
uv run python sample_table.py \
  --input-dir data/raw/2026/Q1 \
  --output-dir data/sample_summary/2026/Q1 \
  --month-group '1-2月=01-02' \
  --month-group '3月=03'
```

分组格式：

- `列名=月份`
- `列名=年份:月份`

例如：

- `1-2月=01-02`
- `3月=03`
- `2025年12月=2025:12`

## 6. 主流程产物关系

一次完整运行会生成：

```text
data/satisfaction_detail/{year}/{batch}/
  客户类型分项统计 xlsx

data/satisfaction_summary/{year}/{batch}/
  {batch}客户类型满意度汇总表.xlsx

data/sample_summary/{year}/{batch}/
  {batch}客户类型样本统计表.xlsx
```

满意度汇总表依赖分项统计结果；样本统计表直接依赖原始数据，因此两者可用于交叉检查：

- 满意度分数是否符合分项文件
- 样本数量是否符合原始记录
- 年月分组是否符合原始数据的 `年份` / `月份` 标记

# 客户类型汇总表

## 目的

把 `survey_stats.py` 产出的单群体统计结果 `xlsx` 再汇总成截图里的“杭博客户类型满意度情况表”。

输入要求：
- 每个 `xlsx` 为一个客户群体统计结果
- 第一行表头至少包含 `指标`、`满意度`
- 总体行和一级指标行建议直接使用 `survey_stats.py` 导出的样式，这样能稳定识别“总体/一级指标/明细指标”

脚本入口：
- [summary_table.py](/Users/zhangqijin/PycharmProjects/hangbo/summary_table.py)

## 大类与样本类型

### 一、会展客户

- 展览活动主（承）办：来源客户群体 `展览主承办`
- 参展商：来源客户群体 `参展商`
- 专业观众：来源客户群体 `专业观众`
- 会展服务商：来源客户群体 `会展服务商`
- 会议活动主（承）办：来源客户群体 `会议主承办`
- 参会客户：来源客户群体 `参会人员`

### 二、餐饮客户

- 商务简餐：来源客户群体 `商务简餐`
- 特色美食廊：来源客户群体 `特色美食廊`
- 宴会：来源客户群体 `宴会`
- 婚宴：来源客户群体 `婚宴`
- 自助餐：来源客户群体 `自助餐`

### 三、G20峰会体验馆

- 旅行社工作人员：来源客户群体 `旅行社工作人员`
- 游客：来源客户群体 `游客`

### 四、专项调研

- 当前保留空行，不做数据匹配；原先不适用的灰格位置统一显示 `--`

### 五、酒店客户

- 散客：来源客户群体 `散客`
- 住宿团队：来源客户群体 `住宿团队`
- 酒店会议活动主（承）办：来源客户群体 `酒店会议主承办`
- 酒店参会客户：来源客户群体 `酒店参会客户`，兼容别名 `酒店参会人员`
- 酒店餐饮客户：来源客户群体 `酒店宴会`、`酒店自助餐`，按列做算术平均

## 列映射规则

### 总分

- 取单群体统计结果中的总体满意度

### 产品服务

- 会展客户、酒店会议客户：优先取一级指标 `会展服务`，兼容旧输出里的 `会场服务`
- 酒店散客、住宿团队：取一级指标 `入住服务`
- G20峰会体验馆：取一级指标 `旅游服务`

### 硬件设施

- 取一级指标 `硬件设施`

### 配套服务

- 取一级指标 `配套服务`
- 例外：`特色美食廊` 在汇总表中不统计 `配套服务`，对应原灰格位置显示 `--`

### 智慧场馆/服务

- 优先取一级指标 `智慧场馆`
- 如果没有，则取一级指标 `智慧服务`

### 餐饮服务

- 优先取一级指标 `餐饮服务`
- 如果没有一级指标 `餐饮服务`，则回退到明细指标 `餐饮服务`
- 这个回退主要用于会展类/酒店参会类报表，因为它们通常只有 `配套服务` 一级指标，`餐饮服务` 只是其下的一项明细

## 输出规则

- 输出文件默认为 `客户类型满意度汇总表.xlsx`
- 顶部标题为 `杭博客户类型满意度情况表`
- 左侧客户大类按截图顺序合并单元格
- 数值单元格统一按两位小数显示
- 均值计算使用 `Decimal` 后再按 Excel 风格四舍五入，避免 `9.505` 这类边界值因为浮点误差被算成 `9.50`
- 原灰格/不适用的列统一显示 `--`
- 其他本来就是空白但不是灰格的位置保持空白
- 底部 `总分` 行按各样本类型行做列平均，空值不参与计算

## 运行示例

```bash
uv run python summary_table.py \
  --input-dir 输出结果 \
  --output-dir 汇总结果
```

# 问卷期次列预处理

## 背景

部分新版问卷源文件会在 `问卷数据` sheet 的第三列新增期次列，常见值如：

- `一期`
- `二期`
- `三期`

这会让原本依赖固定 Excel 列号的分组列和指标列整体右移，导致 [survey_stats.py](/Users/zhangqijin/PycharmProjects/hangbo/survey_stats.py) 按旧模板取值时错位。

## 当前处理规则

[phase_column_preprocess.py](/Users/zhangqijin/PycharmProjects/hangbo/phase_column_preprocess.py) 现在是独立的预处理模块，[survey_stats.py](/Users/zhangqijin/PycharmProjects/hangbo/survey_stats.py) 在正式统计前会直接调用它。处理逻辑如下：

- 如果第三列没有检测到 `X期` 这类值，不做任何处理
- 如果第三列没有检测到，但别的列检测到 `一期`、`二期` 这类值，会提示该列名、列位置和示例值，提醒“可能已经处理过”
- 如果第三列检测到 `一期`、`二期` 等期次标记，就把整列移动到当前数据最后一列
- 移动后会直接保存回原始 Excel，再基于保存后的结果继续计算

## 单独运行

可以把它单独当脚本使用：

```bash
uv run python phase_column_preprocess.py 'datas/3月/展览.xlsx'
```

也可以一次传多个文件：

```bash
uv run python phase_column_preprocess.py datas/3月/*.xlsx
```

如果不是默认 sheet，可以额外指定：

```bash
uv run python phase_column_preprocess.py 'datas/3月/展览.xlsx' --sheet-name '问卷数据'
```

## 终端提示

单独运行 [phase_column_preprocess.py](/Users/zhangqijin/PycharmProjects/hangbo/phase_column_preprocess.py) 时，终端会逐文件输出以下类型的提示：

- 开始检查文件
- 文件不存在
- 缺少指定 sheet
- 文件列数不足，未发现第三列
- 第三列未检测到 `一期` / `二期` 等期次标记，但别的列发现了符合特征的值，提示“可能已经处理过”
- 整张表未发现期次特征列，无需处理
- 第三列检测到期次标记，已移动到最后一列并保存
- 全部文件处理结束后的汇总统计，会按以下类别分组列出文件：
- 成功处理
- 未处理，疑似已处理过
- 未处理，不含期次特征列
- 未处理，列数不足
- 失败
- 最后一行会再输出一条总括总结，汇总本次检查的总文件数和各分类数量

当前 `survey_stats.py` 的终端输出以文件处理进度为主，不再额外打印计算结果表格；这条预处理提示会和对应文件的进度提示一起出现。

## 影响

- 这个预处理对单任务模式、配置批量模式、兼容旧版三输入模式都会自动生效
- 同一个文件如果已经处理过，再次运行时不会重复移动
- `survey_stats.py` 内部调用预处理时，只有真正移动了第三列才会额外打印提示；“疑似已处理过”只在单独运行预处理脚本时显示

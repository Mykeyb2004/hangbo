# 新数据分析流程说明

本文档面向当前仓库里的**新主流程**，重点说明：

- 如何准备原始数据目录
- 如何进行预查错
- 如何给数据集补写 `年份` / `月份`
- 多月数据如何合并
- 最后如何一键跑到 PPT

适用入口：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

---

## 1. 新流程的目标

当前推荐的数据分析流程是：

`原始数据目录 -> 预查错 -> 必要时人工修正 -> 满意度分项统计 -> 满意度汇总 -> 样本汇总 -> PPT`

主流程入口固定读取：

- `data/raw/{year}/{batch}`

并固定输出到：

- `data/satisfaction_detail/{year}/{batch}`
- `data/satisfaction_summary/{year}/{batch}`
- `data/sample_summary/{year}/{batch}`
- `data/ppt/{year}/{batch}`

例如：

- 原始数据：`data/raw/2026/3月`
- 分项统计：`data/satisfaction_detail/2026/3月`
- 满意度汇总：`data/satisfaction_summary/2026/3月`
- 样本汇总：`data/sample_summary/2026/3月`
- PPT：`data/ppt/2026/3月/3月满意度报告.pptx`

---

## 2. 目录约定与批次理解

### 2.1 单月批次

单月批次目录形如：

- `data/raw/2026/1月`
- `data/raw/2026/2月`
- `data/raw/2026/3月`

这类批次可以由程序识别为“单个月份”，因此：

- 预查错时，如果缺少 `年份` / `月份` 列，主流程可以自动补齐

### 2.2 合并批次

合并批次目录形如：

- `data/raw/2026/1-2月`
- `data/raw/2026/Q1`

这类目录表示其中的数据**可能来自多个自然月**，因此：

- 程序不能自动推断每一行到底属于哪一个月
- 如果缺少 `年份` / `月份` 列，主流程会阻断，要求先补齐再继续

### 2.3 标准来源文件

主流程预期的标准来源文件包括：

- `展览.xlsx`
- `会议.xlsx`
- `酒店.xlsx`
- `餐饮.xlsx`
- `会展服务商.xlsx`
- `旅游.xlsx`

说明：

- 批次目录里**不要求每次都存在全部文件**
- 某个月份缺少某些客户分组来源文件，是允许的
- 但如果整个批次目录里一个标准来源文件都没有，主流程会直接阻断

---

## 3. 预查错怎么做

### 3.1 预查错在主流程中的位置

执行：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

后，程序会先做预查错，再决定是否进入后续统计。

主流程执行顺序是：

1. 预查错
2. 如有阻断问题，停下来等待人工修改
3. 修改后重新检查
4. 通过后继续生成分项统计、汇总、样本、PPT

### 3.2 预查错检查哪些内容

当前预查错主要检查四类问题：

#### A. 原始批次目录是否存在

例如：

- `data/raw/2026/3月`
- `data/raw/2026/Q1`

如果批次目录不存在，会直接阻断。

#### B. 是否至少存在一个标准来源文件

如果批次目录存在，但：

- `展览.xlsx`
- `会议.xlsx`
- `酒店.xlsx`
- `餐饮.xlsx`
- `会展服务商.xlsx`
- `旅游.xlsx`

这几类文件一个都没有，也会直接阻断。

但要注意：

- **只缺某几个文件，不阻断，也不作为异常提示**
- 因为某个月份里没有某些客户分组来源，本来就是正常情况

#### C. 是否具备 `年份` / `月份` 列

- 单月批次：
  - 缺失时只给 warning
  - 后续主流程会自动补齐
- 合并批次：
  - 缺失时直接阻断
  - 需要先手动补齐或重新按规范合并

#### D. 是否存在未映射标签记录

预查错还会检查：

- 原始问卷中的辅助标签 + 数据标签组合
- 是否能落到当前客户分类规则

如果发现未映射记录，会阻断主流程，并生成日志：

- `logs/pipeline/{year}/{batch}/unmapped_customer_records.log`

这一步的目的，是先把“数据标签脏数据”拦在统计之前。

### 3.3 预查错后如何继续

如果预查错出现阻断问题，主流程会在终端提示你先修改原始数据。

修改完成后，在终端输入：

- `y`
- `yes`
- `继续`

即可重新检查并继续主流程。

## 4. 给数据集添加年份 / 月份标记

### 4.1 这个动作是做什么的

`fill_year_month_columns.py` 的作用是：

- 遍历目录中的 `xlsx`
- 只处理 `问卷数据` sheet
- 给每个文件补写 `年份`、`月份` 两列
- 如果这两列已存在，则覆盖原值

它是“原始数据增强”工具，适合在多月合并前做准备。

### 4.2 它的写入方式

这个脚本是**整文件批量写值**，不是逐行识别月份。

也就是说：

- 传入 `--year 2026 --month 3`
- 该文件中所有数据行都会被写成：
  - `年份 = 2026`
  - `月份 = 3`

因此它适合：

- 单月目录

不适合直接处理：

- `1-2月`
- `Q1`

这类一份文件中混有多个月份数据的目录。

### 4.3 推荐用法

#### 处理单月目录

```bash
uv run python fill_year_month_columns.py \
  --input-dir 'data/raw/2026/3月' \
  --year '2026' \
  --month '3'
```

如果要处理 `1月`、`2月`、`3月` 三个单月目录，推荐分别执行：

```bash
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/1月' --year '2026' --month '1'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/2月' --year '2026' --month '2'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/3月' --year '2026' --month '3'
```

### 4.4 主流程里的自动补齐规则

新主流程只会在**单月批次**下自动触发这一步。

例如：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

如果 `data/raw/2026/3月` 缺少 `年份` / `月份`，程序会自动补：

- `年份 = 2026`
- `月份 = 3`

但如果是：

```bash
uv run python main_pipeline.py --year 2026 --batch Q1
```

程序不会自动猜测每一行属于哪个月，而是会要求你先准备好正确的年月信息。

### 4.5 一个重要边界

如果你手里只有：

- `1-2月`
- `Q1`

这种已经混合好的历史批次文件，而且里面没有 `年份` / `月份` 列，那么：

- 不能直接用 `fill_year_month_columns.py` 一把写成某一个月
- 否则会把混合月份数据全部盖成同一个月份，造成错误

这类情况正确做法通常是：

1. 找回原始单月目录
2. 先按单月补年月
3. 再重新做多月合并

## 5. 多月如何合并

### 5.1 使用哪个脚本

多月合并由：

- `merge_questionnaire_workbooks.py`

负责。

### 5.2 合并逻辑是什么

这个脚本的核心规则是：

#### A. 按文件名分组

如果多个输入目录里都有：

- `展览.xlsx`
- `会议.xlsx`
- `餐饮.xlsx`

那么它会把同名文件归为一组分别处理。

#### B. 只读 `问卷数据` sheet

如果某个文件没有这个 sheet，会跳过并报错。

#### C. 相同语义列会自动对齐

脚本会忽略题号前缀，例如：

- `Q1-调研类别`
- `Q2-调研类别`

会被视为同一语义列。

因此，不同月份问卷题号略有变化时，仍然可以对齐到统一表头。

#### D. 行是直接追加的

不同来源目录中的数据行，会按统一表头对齐后，直接依次追加到输出文件中。

它**不会**自动做这些事情：

- 不按 `提交序号` 去重
- 不自动选择“更新版本”记录
- 不自动修正客户标签
- 不自动判断哪条记录应该保留

所以它本质上是：

- **同名问卷文件的表头对齐 + 行拼接工具**

### 5.3 推荐的多月合并顺序

如果你要把多个月份合并成季度批次，推荐顺序如下：

#### 第一步：先对每个单月目录补年份 / 月份

```bash
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/1月' --year '2026' --month '1'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/2月' --year '2026' --month '2'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/3月' --year '2026' --month '3'
```

#### 第二步：执行多月合并

例如把 `1月 + 2月 + 3月` 合并到 `Q1`：

```bash
uv run python merge_questionnaire_workbooks.py \
  --input-dir 'data/raw/2026/1月' \
  --input-dir 'data/raw/2026/2月' \
  --input-dir 'data/raw/2026/3月' \
  --output-dir 'data/raw/2026/Q1'
```

也可以按你当前历史数据组织方式，把：

- `1-2月`
- `3月`

合并为 `Q1`，但前提是 `1-2月` 里的每条记录已经具备正确的 `年份` / `月份` 信息。

### 5.4 合并完成后如何进入新主流程

一旦合并结果已经放到：

- `data/raw/2026/Q1`

就可以直接运行：

```bash
uv run python main_pipeline.py --year 2026 --batch Q1
```

这里要特别注意：

- 新主流程不会自动帮你执行“把 1-2月 + 3月 合成 Q1”
- `Q1` 必须先作为一个**准备好的原始批次目录**存在

## 6. 如何最终生成 PPT

### 6.1 一键主流程生成

推荐方式是直接跑主流程：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

或者：

```bash
uv run python main_pipeline.py --year 2026 --batch Q1
```

主流程通过预查错后，会自动依次完成：

1. 满意度分项统计
2. 满意度汇总
3. 样本汇总
4. PPT 生成

最终 PPT 会输出到：

- `data/ppt/{year}/{batch}/{batch}满意度报告.pptx`

例如：

- `data/ppt/2026/3月/3月满意度报告.pptx`
- `data/ppt/2026/Q1/Q1满意度报告.pptx`

### 6.2 主流程内部各步的数据依赖

主流程内部依赖关系如下：

#### A. 分项统计

读取：

- `data/raw/{year}/{batch}`

生成：

- `data/satisfaction_detail/{year}/{batch}`

#### B. 满意度汇总

读取：

- `data/satisfaction_detail/{year}/{batch}`

生成：

- `data/satisfaction_summary/{year}/{batch}`

#### C. 样本汇总

读取：

- `data/raw/{year}/{batch}`

生成：

- `data/sample_summary/{year}/{batch}`

#### D. PPT

读取：

- `data/satisfaction_detail/{year}/{batch}`

生成：

- `data/ppt/{year}/{batch}/{batch}满意度报告.pptx`

也就是说：

- PPT 的直接输入是“分项统计结果”
- 但从业务流程顺序上，它放在汇总和样本之后作为最终交付物

## 7. 推荐操作模板

### 7.1 只处理单月

适用场景：

- 例如只处理 `3月`

推荐操作：

```bash
uv run python main_pipeline.py --year 2026 --batch 3月
```

说明：

- 如果缺少 `年份` / `月份` 列，主流程会自动补齐
- 预查错通过后直接一路跑到 PPT

### 7.2 先准备季度，再跑季度主流程

适用场景：

- 例如要重新生成 `Q1`

推荐操作：

```bash
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/1月' --year '2026' --month '1'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/2月' --year '2026' --month '2'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/3月' --year '2026' --month '3'

uv run python merge_questionnaire_workbooks.py \
  --input-dir 'data/raw/2026/1月' \
  --input-dir 'data/raw/2026/2月' \
  --input-dir 'data/raw/2026/3月' \
  --output-dir 'data/raw/2026/Q1'

uv run python main_pipeline.py --year 2026 --batch Q1
```

### 7.3 已有历史混合批次

适用场景：

- 已经存在 `data/raw/2026/1-2月`
- 已经存在 `data/raw/2026/Q1`

推荐操作：

1. 先确认这些历史混合文件里是否已经带有正确的 `年份` / `月份`
2. 再直接运行主流程

例如：

```bash
uv run python main_pipeline.py --year 2026 --batch 1-2月
uv run python main_pipeline.py --year 2026 --batch Q1
```

如果混合批次缺少 `年份` / `月份`，不要直接整文件补写单一月份，而应回到单月源数据重新整理。

## 8. 一句话记忆版

如果把新流程压缩成一句话，可以记成：

### 单月批次

`把原始文件放进 data/raw/{year}/{batch} -> 跑 main_pipeline.py -> 程序自动预查错并完成后续输出`

### 多月批次

`先在单月目录补年份/月度 -> 再用 merge_questionnaire_workbooks.py 合并 -> 把结果放进 data/raw/{year}/{batch} -> 再跑 main_pipeline.py`

### 最终交付

`主流程跑通后，PPT 会自动生成到 data/ppt/{year}/{batch}`

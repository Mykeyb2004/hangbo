# 批次管理与配置持久化

## 目标

这一版先把 GUI 从“单次操作窗口”升级为“可反复使用的批次工作台”：

- 支持把当前界面配置保存为命名批次
- 支持在多个已保存批次之间切换
- 支持删除不再使用的批次配置
- 支持关闭 GUI 后自动恢复最近一次会话

## 当前实现

### 1. 顶部批次管理区

GUI 顶部增加了一组批次管理控件：

- `批次名称`
- `已保存批次`
- `加载`
- `新建`
- `保存批次`
- `删除批次`

交互约定：

- `保存批次`：按当前 `批次名称` 保存或覆盖同名批次
- `加载`：把下拉列表中的已保存批次加载到当前界面
- `新建`：基于当前配置复制一份草稿，并把批次名称重置为 `未命名批次`
- `删除批次`：只删除保存下来的 GUI 配置文件，不删除原始数据、统计结果或 PPT

### 2. 存储目录

批次和会话文件都保存在项目内：

```text
logs/
  gui_profiles/
    last_session.toml
    batches/
      batch_<批次名称utf8十六进制>.toml
```

说明：

- `last_session.toml` 保存最近一次关闭 GUI 时的表单状态
- `batches/` 下每个文件对应一个“已保存批次”
- 文件名不直接使用中文名，而是使用 UTF-8 十六进制编码，避免不同平台上的文件名兼容问题

### 3. 会话恢复

窗口关闭时，GUI 会自动把当前表单状态写入 `last_session.toml`。

下次启动时：

1. 优先读取 `last_session.toml`
2. 恢复上次关闭时的表单内容
3. 如果上次会话关联的是某个已保存批次，则同时在下拉框中选中该批次

这意味着即使用户还没点“保存批次”，也能恢复上次未完成的编辑状态。

## 配置字段范围

当前会被持久化的字段包括：

- 批次名称
- 处理模式
- 单月输入目录
- 多月合并输入目录列表
- 合并输出目录
- sheet 名称
- 年份、月份
- 分项统计输出目录
- 计算口径
- 输出格式
- 汇总输出目录
- 汇总输出文件名
- PPT 模板路径
- PPT 输出路径
- `section_mode`
- 空值显示文案

当前不会持久化的运行时状态：

- 实时日志内容
- 主流程执行中的状态机
- 分项统计扫描结果表格
- 已勾选客群结果

这些状态会在重新启动后重新扫描或重新生成，避免把旧数据误当成当前数据。

## 当前边界

这一版先保证“可用”和“稳定”：

- 批次重命名通过“修改批次名称后重新保存，再删除旧批次”完成
- `新建` 默认复制当前配置做草稿，便于按月复用
- 任务执行中禁止加载/新建/删除批次，避免运行上下文被中途切换

后续如果继续增强，优先级建议是：

1. 增加“另存为”弹窗
2. 给批次列表增加最近修改时间
3. 支持导入/导出批次模板
4. 把批次列表做成单独页面，支持备注和分类

# 用户运行用例故事：4月数据与半年数据处理

本文档面向实际运行人员，以 `2026 年 4 月` 数据为例，说明从原始数据预查错、人工干预未映射记录、补写年月，到最终生成 PPT 的完整操作。

文末补充 `1-6 月` 半年数据合并与生成 PPT 的用例。

> 本文按当前推荐的新主流程编写：原始数据放在 `data/raw/{year}/{batch}`，所有命令均在项目根目录执行，并统一使用 `uv run`。

---

## 1. 用户故事

作为一名数据处理用户，我希望把某个月或某个时间段的问卷 Excel 放到约定目录后，先发现会影响统计的问题，能够根据日志定位并修正未映射记录，再一键生成满意度分项、汇总、样本统计和 PPT，并且可以选择是否让 LLM 自动写每页的数据分析描述。

成功结果包括：

- 原始数据通过预查错，没有阻断问题。
- 原始问卷具备正确的 `年份` / `月份` 字段。
- 满意度分项结果生成到 `data/satisfaction_detail/{year}/{batch}`。
- 满意度汇总表生成到 `data/satisfaction_summary/{year}/{batch}`。
- 样本统计表生成到 `data/sample_summary/{year}/{batch}`。
- PPT 生成到 `data/ppt/{year}/{batch}`。

## 2. 用例一：处理 2026 年 4 月数据

### 2.1 前置准备

用户先把 4 月原始问卷文件放到：

```text
data/raw/2026/4月
```

目录中可以包含以下标准来源文件的一部分或全部：

```text
展览.xlsx
会议.xlsx
酒店.xlsx
餐饮.xlsx
会展服务商.xlsx
旅游.xlsx
```

注意：

- 不要求每个月都有全部来源文件；某个月没有某类客户数据是正常情况。
- 但目录里至少要有一个标准来源文件，否则主流程会阻断。
- 每个 Excel 需要有 `问卷数据` sheet。
- 运行脚本前请先关闭正在编辑的 Excel 文件，避免保存失败或读写冲突。

### 2.2 首先怎么预查错

当前完整预查错内置在主流程第一步中。处理 4 月时执行：

```bash
uv run python main_pipeline.py --year 2026 --batch 4月
```

主流程会先检查：

- `data/raw/2026/4月` 是否存在。
- 是否至少存在一个标准来源文件。
- 标准来源文件中是否有 `问卷数据` sheet。
- 是否缺少 `年份` / `月份` 列。
- 是否存在未被客户类别映射规则覆盖的记录。

预查错日志会写入：

```text
logs/pipeline/2026/4月/precheck.log
logs/pipeline/2026/4月/unmapped_customer_records.log
logs/pipeline/2026/4月/pipeline.log
```

如果只是想提前核查“未映射客户记录”，不想立刻进入后续统计，可以单独运行：

```bash
uv run python check_unmapped_customer_records.py \
  --input-dir 'data/raw/2026/4月' \
  --log-file 'logs/pipeline/2026/4月/manual_unmapped_check.log'
```

这条命令只做客户标签映射核查，不会生成分项统计、汇总表或 PPT。

### 2.3 发现未映射记录后，如何人工干预

如果终端提示类似：

```text
[PRECHECK] 发现阻断问题：
- 发现 3 条未映射标签记录，请查看：logs/pipeline/2026/4月/unmapped_customer_records.log
请先修改原始数据目录：data/raw/2026/4月
修改完成后输入 y / yes / 继续；输入 stop 终止：
```

用户需要按以下步骤处理。

#### 第一步：打开未映射日志

打开：

```text
logs/pipeline/2026/4月/unmapped_customer_records.log
```

重点查看日志里的：

- 来源文件名，例如 `会议.xlsx`、`餐饮.xlsx`。
- Excel 行号。
- 数据标签列和值。
- 辅助标签列和值。
- 未命中映射的原因。

日志中的 Excel 行号可以直接对应到原始 Excel 的行号。

#### 第二步：回到原始 Excel 修改具体行

根据日志提示，打开对应文件，例如：

```text
data/raw/2026/4月/会议.xlsx
```

进入 `问卷数据` sheet，跳到日志提示的 Excel 行号，检查对应的辅助标签列、数据标签列。

常见处理方式有两类。

**情况 A：原始记录填错、写错或选错了标签**

这时应该修改原始 Excel 里的错误单元格，让它回到既有映射规则能识别的标签。

示例：

- `展览.xlsx` 通常按 `C=展览` + `E=展览主承办/参展商/专业观众` 识别。
- `会议.xlsx` 通常按 `C=会议/酒店会议` + `E=会议主承办/参会人员/酒店会议主承办/酒店参会客户` 识别。
- `餐饮.xlsx` 通常按 `C=餐饮/酒店餐饮` + `D=商务简餐/特色美食廊/宴会/婚宴/自助餐/酒店宴会/酒店自助餐` 识别。
- `酒店.xlsx` 通常按 `C=散客/住宿团队` 识别。
- `旅游.xlsx` 通常按 `C=游客/旅行社工作人员` 识别。
- `会展服务商.xlsx` 通常按 `D=会展服务商` 识别。

修改后保存 Excel，并关闭文件。

**情况 B：这是一个业务确认要新增的客户标签**

这时不要为了通过程序而强行改成旧标签。应先确认它是否确实应该成为新的映射关系。

当前业务映射依据是：

```text
datas/customer_mapping.xlsx
```

当前程序实际执行的规则是：

```text
survey_customer_category_rules.py
```

如果需要新增映射关系，建议由规则维护人完成：

1. 先在 `datas/customer_mapping.xlsx` 中补充业务映射依据。
2. 再同步更新 `survey_customer_category_rules.py` 中的可执行规则。
3. 运行映射相关测试，确认规则、顺序和核查逻辑一致。

建议测试命令：

```bash
uv run python -m unittest \
  tests/test_survey_customer_category_rules.py \
  tests/test_survey_customer_mappings.py \
  tests/test_check_unmapped_customer_records.py
```

如果运行用户没有维护规则的权限，应把日志、来源文件名、Excel 行号和业务标签值发给规则维护人处理。

#### 第三步：让主流程重新检查

如果正在运行 `main_pipeline.py`，修完 Excel 并保存关闭后，在终端输入：

```text
y
```

也可以输入：

```text
yes
继续
```

主流程会重新预查错。只要仍存在阻断问题，就会继续停下来等待人工修改。

如果只是单独运行 `check_unmapped_customer_records.py`，则重新执行核查命令，直到日志中显示未发现未映射记录。

### 2.4 如何给 4 月数据添加年月

对于 `4月` 这种单月批次，主流程可以自动补写 `年份` / `月份`。

也就是说，直接执行：

```bash
uv run python main_pipeline.py --year 2026 --batch 4月
```

如果预查错发现标准来源文件缺少 `年份` / `月份` 列，会提示 warning，然后自动把 4 月目录中所有数据行补成：

```text
年份 = 2026
月份 = 4
```

如果用户希望在正式主流程前手工补写，也可以先执行：

```bash
uv run python fill_year_month_columns.py \
  --input-dir 'data/raw/2026/4月' \
  --year '2026' \
  --month '4'
```

注意：

- 这个脚本会遍历目录中的 `xlsx` 文件。
- 只写入 `问卷数据` sheet。
- 如果已有 `年份` / `月份` 列，会覆盖原值。
- 它适合单月目录，不适合直接处理 `1-6月` 这种混有多个月份的合并目录。

### 2.5 如何走完后面的 PPT 生成

当预查错通过后，主流程会自动继续执行：

1. 生成满意度分项统计。
2. 生成满意度汇总表。
3. 生成样本统计表。
4. 生成 PPT。

继续使用同一条命令即可：

```bash
uv run python main_pipeline.py --year 2026 --batch 4月
```

4 月的主要输出位置为：

```text
data/satisfaction_detail/2026/4月
data/satisfaction_summary/2026/4月/4月客户类型满意度汇总表.xlsx
data/sample_summary/2026/4月/4月客户类型样本统计表.xlsx
data/ppt/2026/4月/4月满意度报告.pptx
```

如果只想在已有分项统计结果基础上重新生成 PPT，可以直接运行：

```bash
uv run python generate_ppt.py \
  --template-path templates/template.pptx \
  --input-dir 'data/satisfaction_detail/2026/4月' \
  --output-ppt 'data/ppt/2026/4月/4月满意度报告.pptx'
```

直接参数模式默认不会启用 LLM 备注分析；如需启用 LLM，建议使用配置文件方式。

### 2.6 如何开关 LLM 写数据分析描述

LLM 数据分析描述会为每页 PPT 生成一段中文分析文字，写入备注页；如果启用了图表页，也会优先复用到图表页右侧文字框。

#### 主流程方式：修改或复制 `pipeline.defaults.toml`

主流程读取：

```text
pipeline.defaults.toml
```

其中开关位置为：

```toml
[ppt.llm_notes]
enabled = true
```

如果要启用 LLM：

1. 保持 `enabled = true`。
2. 准备 `.env`：

```bash
cp .env.example .env
```

3. 在 `.env` 中填写真实连接信息，例如：

```env
OPENAI_API_KEY=your-api-key
OPENAI_BASE_URL=https://your-openai-compatible-endpoint/v1
OPENAI_MODEL=gpt-4.1-mini
OPENAI_TEMPERATURE=0.4
```

4. 重新运行主流程：

```bash
uv run python main_pipeline.py --year 2026 --batch 4月
```

如果要关闭 LLM，建议不要直接改公共默认配置，而是复制一份临时配置：

```bash
cp pipeline.defaults.toml pipeline.4月.no-llm.toml
```

把临时配置中的开关改成：

```toml
[ppt.llm_notes]
enabled = false
```

然后运行：

```bash
uv run python main_pipeline.py \
  --year 2026 \
  --batch 4月 \
  --config pipeline.4月.no-llm.toml
```

#### PPT 配置文件方式：修改 `[llm_notes]`

如果直接使用 `generate_ppt.py --config ...` 生成 PPT，则开关位于 PPT 作业配置文件的：

```toml
[llm_notes]
enabled = false
```

改成 `true` 即启用；保持 `false` 即关闭。

示例运行：

```bash
uv run python generate_ppt.py --config ppt_job.example.toml
```

启用 LLM 前同样需要先准备 `.env`。如果启用了 LLM 但 `.env` 或接口配置不可用，PPT 生成会在 LLM 调用阶段失败或中断。

## 3. 用例二：合并 1-6 月数据形成半年数据并生成 PPT

### 3.1 用户故事

作为一名数据处理用户，我希望把 `1月` 到 `6月` 的原始问卷合并成一个半年批次，保留每行原本所属的年份和月份，再按同一套主流程生成半年满意度结果和 PPT。

半年批次建议命名为：

```text
1-6月
```

最终原始数据目录为：

```text
data/raw/2026/1-6月
```

### 3.2 合并前先补每个月的年月

半年数据是混合批次，主流程无法自动判断每一行来自哪个月。

因此必须先在每个单月目录补好 `年份` / `月份`，再合并。

推荐逐月执行：

```bash
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/1月' --year '2026' --month '1'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/2月' --year '2026' --month '2'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/3月' --year '2026' --month '3'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/4月' --year '2026' --month '4'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/5月' --year '2026' --month '5'
uv run python fill_year_month_columns.py --input-dir 'data/raw/2026/6月' --year '2026' --month '6'
```

如果确认 `Q1` 已经是准备好的季度目录，并且里面每行已经有正确 `年份` / `月份`，也可以使用 `Q1 + 4月 + 5月 + 6月` 的合并方式。

但不要同时把 `Q1` 和 `1月/2月/3月` 放进同一次合并，否则会重复计算 1-3 月数据。

### 3.3 合并 1-6 月同名 Excel

推荐从 6 个单月目录合并：

```bash
uv run python merge_questionnaire_workbooks.py \
  --input-dir 'data/raw/2026/1月' \
  --input-dir 'data/raw/2026/2月' \
  --input-dir 'data/raw/2026/3月' \
  --input-dir 'data/raw/2026/4月' \
  --input-dir 'data/raw/2026/5月' \
  --input-dir 'data/raw/2026/6月' \
  --output-dir 'data/raw/2026/1-6月'
```

如果使用已准备好的 `Q1`：

```bash
uv run python merge_questionnaire_workbooks.py \
  --input-dir 'data/raw/2026/Q1' \
  --input-dir 'data/raw/2026/4月' \
  --input-dir 'data/raw/2026/5月' \
  --input-dir 'data/raw/2026/6月' \
  --output-dir 'data/raw/2026/1-6月'
```

合并脚本会按同名文件合并，例如把各月的 `展览.xlsx` 合并成半年目录下的一个 `展览.xlsx`。

合并完成后，用户应确认：

- `data/raw/2026/1-6月` 已生成。
- 半年目录中存在需要统计的标准来源文件。
- 每个合并后的文件仍有 `问卷数据` sheet。
- 合并后的数据保留了各行原本的 `年份` / `月份`。

### 3.4 对半年数据做预查错

可以先单独核查半年目录是否存在未映射记录：

```bash
uv run python check_unmapped_customer_records.py \
  --input-dir 'data/raw/2026/1-6月' \
  --log-file 'logs/pipeline/2026/1-6月/manual_unmapped_check.log'
```

也可以直接进入主流程：

```bash
uv run python main_pipeline.py --year 2026 --batch 1-6月
```

注意：

- `1-6月` 是合并批次，不是单月批次。
- 如果合并后的文件缺少 `年份` / `月份` 列，主流程会阻断。
- 此时不要对 `data/raw/2026/1-6月` 直接统一补 `月份=6`，否则会把 1-5 月行也错误改成 6 月。
- 正确做法是回到单月目录补好各自年月，然后重新合并。

### 3.5 处理半年未映射记录

半年批次的未映射处理方式与 4 月一致：

1. 查看日志：

```text
logs/pipeline/2026/1-6月/unmapped_customer_records.log
```

2. 根据来源文件名、Excel 行号、辅助标签和值、数据标签和值，打开：

```text
data/raw/2026/1-6月/{来源文件名}
```

3. 如果是原始标签错误，直接修正半年目录中的错误单元格。
4. 如果是业务新增标签，走映射关系维护流程。
5. 保存关闭 Excel 后，在主流程终端输入 `y` / `yes` / `继续` 重新检查。

如果希望源头数据也保持一致，建议同时回改对应单月目录，再重新合并半年目录。

### 3.6 生成半年分项、汇总、样本和 PPT

半年预查错通过后，主流程会自动完成后续步骤：

```bash
uv run python main_pipeline.py --year 2026 --batch 1-6月
```

半年主要输出位置为：

```text
data/satisfaction_detail/2026/1-6月
data/satisfaction_summary/2026/1-6月/1-6月客户类型满意度汇总表.xlsx
data/sample_summary/2026/1-6月/1-6月客户类型样本统计表.xlsx
data/ppt/2026/1-6月/1-6月满意度报告.pptx
```

LLM 开关与 4 月完全相同：

- 主流程使用 `pipeline.defaults.toml` 中的 `[ppt.llm_notes].enabled`。
- 需要启用 LLM 时准备 `.env`。
- 不需要 LLM 时使用一份 `enabled = false` 的配置文件，并通过 `--config` 指定。

示例：

```bash
uv run python main_pipeline.py \
  --year 2026 \
  --batch 1-6月 \
  --config pipeline.半年.no-llm.toml
```

## 5. 验收标准

运行完成后，用户可以按以下标准确认结果：

- `logs/pipeline/{year}/{batch}/precheck.log` 中没有阻断问题。
- `logs/pipeline/{year}/{batch}/unmapped_customer_records.log` 中未发现未映射记录。
- `data/satisfaction_detail/{year}/{batch}` 中有各客户类型分项 Excel。
- `data/satisfaction_summary/{year}/{batch}` 中有客户类型满意度汇总表。
- `data/sample_summary/{year}/{batch}` 中有客户类型样本统计表。
- `data/ppt/{year}/{batch}/{batch}满意度报告.pptx` 可以正常打开。
- 若启用 LLM，PPT 备注页中能看到自动生成的数据分析描述。
- 若关闭 LLM，PPT 仍能生成，但不会自动写备注页分析描述。
